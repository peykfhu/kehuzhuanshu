API 文档

AnyHop AI 兼容 Anthropic / OpenAI API 格式，无缝切换，即改即用。

简介

AnyHop AI 提供统一的 API 网关服务，通过一个 API Key 即可访问 Claude、GPT、Gemini、DeepSeek 等多种主流大模型。接口与 Anthropic / OpenAI 官方完全兼容，现有代码仅需修改 Base URL 即可接入，无需更改任何业务逻辑。

💡 已在使用 Anthropic 或 OpenAI SDK？只需替换 base_url 和 api_key 两个参数即可立即使用 AnyHop AI，无需修改任何其他代码。

认证方式

所有 API 请求都需要在 HTTP Header 中携带 API Key 进行身份认证。AnyHop AI 支持以下两种认证格式：

方式一：Anthropic 原生格式

适用于 Anthropic SDK 或直接调用 Anthropic 兼容端点：

x-api-key: sk-anyhop-xxxxxxxxxxxx

方式二：Bearer Token 格式

适用于 OpenAI SDK 或通用 HTTP 客户端：

Authorization: Bearer sk-anyhop-xxxxxxxxxxxx
⚠️ 安全提醒：请勿在客户端、前端代码或公开仓库中暴露 API Key。如果 Key 不慎泄露，请立即在控制台重新生成。

Base URL

根据你使用的 SDK 类型，选择对应的 Base URL：

兼容格式Base URL适用 SDK
Anthropichttps://anyhop.aianthropic (Python/Node/Go)
OpenAIhttps://anyhop.ai/openai/v1openai (Python/Node)

版本控制

使用 Anthropic 兼容端点时，需要在请求头中指定 API 版本：

anthropic-version: 2023-06-01

当前支持的版本为 2023-06-01，建议始终显式指定版本号以确保接口行为一致。

Messages API

Messages API 是最核心的接口，用于发送消息并获取 AI 模型的回复。完全兼容 Anthropic Messages API 规范。

POST /v1/messages

请求体参数

参数类型必填说明
modelstring是模型 ID，如 claude-sonnet-4-6
messagesarray是对话消息列表，每条消息包含 role 和 content
max_tokensinteger是模型最大输出 Token 数
systemstring | array否系统提示词
temperaturenumber否采样温度 (0-1)，默认 1
streamboolean否是否启用流式输出，默认 false
top_pnumber否Top-P (0-1)
top_kinteger否Top-K
stop_sequencesarray否自定义停止序列
metadataobject否请求元数据

基础请求示例

curl https://anyhop.ai/v1/messages \
  -H "Content-Type: application/json" \
  -H "x-api-key: sk-anyhop-xxxx" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "claude-sonnet-4-6",
    "max_tokens": 1024,
    "messages": [
      {"role": "user", "content": "What is an API gateway?"}
    ]
  }'

响应格式

{
  "id": "msg_01XFDUDYJgAACzvnptvVoYEL",
  "type": "message",
  "role": "assistant",
  "content": [
    {
      "type": "text",
      "text": "An API gateway is a server that acts as an intermediary..."
    }
  ],
  "model": "claude-sonnet-4-6",
  "stop_reason": "end_turn",
  "usage": {
    "input_tokens": 18,
    "output_tokens": 156
  }
}

stop_reason

值说明
end_turn模型正常完成回复
max_tokens达到上限被截断
stop_sequence遇到停止序列
tool_use请求使用工具

多轮对话

通过在 messages 数组中交替排列 user 和 assistant 角色的消息实现多轮对话。

{
  "model": "claude-sonnet-4-6",
  "max_tokens": 1024,
  "messages": [
    {"role": "user", "content": "I'm learning Python, can you help?"},
    {"role": "assistant", "content": "Of course! What topic would you like to start with?"},
    {"role": "user", "content": "How do list comprehensions work?"}
  ]
}
💡 对话消息必须以 user 角色开始，且 user/assistant 需交替出现。每轮对话都会消耗输入 Token。

Streaming（流式输出）

设置 "stream": true 启用 SSE 流式输出，实时逐字获取模型生成的内容。

curl https://anyhop.ai/v1/messages \
  -H "Content-Type: application/json" \
  -H "x-api-key: sk-anyhop-xxxx" \
  -H "anthropic-version: 2023-06-01" \
  -d '{
    "model": "claude-sonnet-4-6",
    "max_tokens": 1024,
    "stream": true,
    "messages": [
      {"role": "user", "content": "Write a short poem about coding"}
    ]
  }'

SSE 事件类型

事件说明
message_start消息开始，包含元数据
content_block_delta增量文本内容
message_delta包含 stop_reason 和 usage
message_stop消息完成
ping心跳保活

OpenAI 兼容接口

AnyHop AI 同时提供完整的 OpenAI Chat Completions 兼容接口，适用于 GPT 系列模型或使用 OpenAI SDK 的场景。

POST /openai/v1/chat/completions 
from openai import OpenAI

client = OpenAI(
    api_key="sk-anyhop-xxxx",
    base_url="https://anyhop.ai/openai/v1"
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Explain microservices architecture"}
    ]
)
print(response.choices[0].message.content)

{

  "system": "You are a professional translator. Translate the user's input to English.",

    {"role": "user", "content": "你好，今天天气怎么样？"}
  ]
}

Tool Use

允许模型调用你预定义的工具/函数来获取外部信息或执行操作。

{
  "model": "claude-sonnet-4-6",
  "max_tokens": 1024,
  "tools": [
    {
      "name": "get_weather",
      "description": "Get current weather for a city",
      "input_schema": {
        "type": "object",
        "properties": {
          "city": {"type": "string", "description": "City name"},
          "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]}
        },
        "required": ["city"]
      }
    }
  ],
  "messages": [
    {"role": "user", "content": "What's the weather in Beijing?"}
  ]
}

Prompt Caching

对于 Claude 模型，支持 Prompt Caching 功能。将重复使用的系统提示词标记为可缓存，可降低约 90% 输入 Token 费用。

{
  "model": "claude-sonnet-4-6",
  "max_tokens": 1024,
  "system": [
    {
      "type": "text",
      "text": "Your long system prompt here...",
      "cache_control": {"type": "ephemeral"}
    }
  ],
  "messages": [{"role": "user", "content": "Hello"}]
}
💡 缓存有效期为 5 分钟，至少需要 1024 Token 才能生效。详见模型广场中的缓存价格。

Claude Code CLI

Claude Code 是 Anthropic 官方推出的命令行编程助手。通过设置环境变量即可让 Claude Code 使用 AnyHop AI 网关。

配置方法

在终端中设置环境变量：

export ANTHROPIC_BASE_URL=https://anyhop.ai
export ANTHROPIC_API_KEY=sk-anyhop-xxxxxxxxxxxx

添加到 shell 配置文件永久生效：

# 添加到 ~/.bashrc 或 ~/.zshrc
echo 'export ANTHROPIC_BASE_URL=https://anyhop.ai' >> ~/.zshrc
echo 'export ANTHROPIC_API_KEY=sk-anyhop-xxxxxxxxxxxx' >> ~/.zshrc
source ~/.zshrc
💡 配置完成后直接运行 claude 命令即可使用。支持所有 Claude 模型。

VSCode - Continue

Continue 是一款流行的 VSCode AI 编程助手插件，支持自定义 API 端点。

配置步骤

1. 在 VSCode 扩展市场搜索并安装 Continue

2. 打开配置文件 ~/.continue/config.json，添加模型配置：

{
  "models": [
    {
      "title": "Claude via AnyHop",
      "provider": "anthropic",
      "model": "claude-sonnet-4-6",
      "apiBase": "https://anyhop.ai",
      "apiKey": "sk-anyhop-xxxxxxxxxxxx"
    }
  ]
}
💡 可添加多个模型配置，在 Continue 面板中自由切换。

错误处理

API 使用标准 HTTP 状态码。

状态码错误类型说明
400invalid_request_error请求参数错误
401authentication_errorAPI Key 无效
403permission_error权限不足
404not_found_error模型不存在
429rate_limit_error请求超限
500api_error服务器错误
529overloaded_error上游过载

速率限制

速率限制取决于你的账户等级。超出限制时返回 429 状态码。

响应头说明
x-ratelimit-limit最大请求数
x-ratelimit-remaining剩余请求数
x-ratelimit-reset重置时间戳
retry-after建议等待秒数

package main

import (
    "context"
    "fmt"
    "github.com/anthropics/anthropic-sdk-go"
    "github.com/anthropics/anthropic-sdk-go/option"
)

func main() {
    client := anthropic.NewClient(
        option.WithAPIKey("sk-anyhop-xxxx"),
        option.WithBaseURL("https://anyhop.ai"),
    )

    message, _ := client.Messages.New(context.TODO(), anthropic.MessageNewParams{
        Model:     anthropic.ModelClaudeSonnet4_6,
        MaxTokens: 1024,
        Messages: []anthropic.MessageParam{
            anthropic.NewUserMessage(anthropic.NewTextBlock("Hello!")),
        },
    })
    fmt.Println(message.Content[0].Text)
}
